# Notebook Achievement Template

Use this template when creating or revising a notebook.

## Opening Template

```markdown
# [Notebook Title]

This notebook explains [idea] by starting from [simple object or action].

By the end, we should understand:

- What [main object] means.
- Why [key split or operation] happens.
- How the small cases force the general pattern.
- What the usual name for this pattern is.
```

## Explanation Spine

```markdown
## Start With The Object

Describe the object without notation first.

## The Smallest Case

Use the smallest meaningful example.

## What Can Happen?

List the real choices, movements, or transformations.

## Build The Pattern

Move from the small case to the general case.

## Edge Cases

Explain what breaks, what becomes zero, and what becomes one.

## The General Rule

Write the formula only after the pattern has been earned.

## The Usual Name

Name the identity, theorem, or rule at the end.

## What This Notebook Achieves

Compress the lesson into a short recap.
```

## Polishing Checklist

- The notebook starts from meaning, not formula.
- Every symbol has been motivated before it is used heavily.
- Edge cases are explained as part of the idea.
- Inline math is used for short expressions inside sentences.
- Display math is used when structure matters.
- Ordinals use superscripts in polished text.
- The theorem or identity name appears after the discovery.
- The final section says what the notebook achieves.
